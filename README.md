# Opioid Pain Management Database System

A relational database project built in Oracle SQL for a fictional medical 
facility's opioid pain management program. Designed to simulate real-world 
healthcare data management workflows including patient tracking, prescription 
records, and risk flagging.

## Project Overview

This project was completed as part of my MS in Healthcare Informatics coursework 
at Grand Canyon University. It covers the full database development lifecycle — 
from entity-relationship modeling through query writing and reporting.

## Tools & Skills Used

- Oracle SQL (DDL, DML, JOINs, GROUP BY, subqueries)
- Entity-Relationship (ER) diagram design
- Database normalization
- Microsoft PowerPoint (project documentation & presentation)

## What's Included

- `opioid_pain_management.pptx` — Full project presentation including schema 
  diagrams, query outputs, and design decisions

## Key Features

- Schema design with constraints for patients, providers, prescriptions, 
  and treatment records
- SQL queries to extract patient treatment history and identify 
  high-risk prescribing patterns
- Summary reporting using aggregations and subqueries
- Formal documentation of methodology and results

## Context

Built to demonstrate foundational healthcare database skills relevant to 
clinical data analyst and healthcare informatics roles.
